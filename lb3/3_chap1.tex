\section{Теоретическое обоснование}

Перед выполнением работы были изучены основные принципы метрической классификации. Метод k ближайших соседей (kNN) является одним из простейших алгоритмов классификации. Он относит объект к тому классу, который является наиболее распространённым среди k его ближайших соседей в пространстве признаков.

Ключевыми этапами реализации kNN являются:
\begin{enumerate}
    \item Выбор метрики для определения расстояния между объектами (например, евклидово расстояние).
    \item Выбор оптимального количества соседей $k$.
    \item Принятие решения о классе объекта на основе голосования соседей.
\end{enumerate}

Оптимизация параметра $k$ и выбор пропорции обучающей и тестовой выборок являются критически важными для построения модели, которая будет хорошо обобщаться на новые данные. Для этого используются такие методы оценки, как \texttt{hold-out} (разделение на обучающую и тестовую выборки) и \texttt{cross-validation} (кросс-валидация).

import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.model_selection import train_test_split, cross_val_score

def age_category(rings):

        return 0 # Молодые
    elif rings < 16:
        return 1 # Взрослые
    else:
        return 2 # Старые

df['AgeCat'] = df['Rings'].apply(age_category)

# Преобразование категориального признака 'Sex' в dummy-переменные
df = pd.get_dummies(df, columns=['Sex'], prefix=['Sex'], drop_first=False)

# Определение признаков (X) и целевой переменной (y)

\textbf{Шаг 4: Построение классификатора с заданием K пользователем.}
Была написана функция, которая принимает на вход параметр \texttt{k\_value} и новые данные, обучает модель на всех данных и возвращает предсказанный класс.

\begin{lstlisting}[language=Python, caption=Функция для предсказания с заданным K]
def predict_with_k(k_value, new_data):
    """
    Обучает kNN классификатор на всех данных и предсказывает класс для new_data.
    """
    model = KNeighborsClassifier(n_neighbors=k_value)
    model.fit(X_scaled, y)
    # new_data должен быть масштабирован тем же scaler'ом
    new_data_scaled = scaler.transform(new_data.reshape(1, -1))
    prediction = model.predict(new_data_scaled)
    return prediction[0]

# Пример использования
k_user = 7
# Возьмем первые 5 признаков случайного объекта из датасета в качестве примера

predicted_class = predict_with_k(k_user, new_example)
print(f"Для K={k_user}, предсказанный класс: {predicted_class}")
\end{lstlisting}

\section{Контрольные вопросы}

\textbf{1. Поясните особенности основных методов метрической классификации: метод ближайшего соседа, метод k ближайших соседей.}
\begin{itemize}
    \item \textbf{Метод ближайшего соседа (1NN):} Это частный случай kNN, где \(k=1\). Объект относится к тому же классу, что и его единственный ближайший сосед. Метод прост, но крайне чувствителен к выбросам и шуму в данных, что часто приводит к переобучению.
    \item \textbf{Метод k ближайших соседей (kNN):} Объект относится к классу, наиболее часто встречающемуся среди его \(k\) ближайших соседей. Выбор \(k\) позволяет сгладить влияние шума и выбросов. Большие значения \(k\) делают границы между классами более гладкими, но могут привести к излишнему обобщению (underfitting), если \(k\) слишком велико.
\end{itemize}

\textbf{2. Поясните основные принципы и этапы реализации метода kNN.}
Основные этапы:
\begin{enumerate}
    \item \textbf{Подготовка данных:} Очистка, обработка пропусков, нормализация/стандартизация признаков, так как метод чувствителен к масштабу.
    \item \textbf{Выбор метрики:} Определение функции расстояния между объектами (например, евклидово, манхэттенское расстояние).
    \item \textbf{Выбор параметра K:} Оптимизация количества соседей, например, с помощью кросс-валидации.
    \item \textbf{Обучение:} В классическом kNN обучение фактически заключается в запоминании обучающей выборки.
    \item \textbf{Классификация нового объекта:}
    \begin{enumerate}
        \item Рассчитать расстояния от нового объекта до всех объектов обучающей выборки.
        \item Отобрать \(k\) объектов с наименьшими расстояниями (ближайших соседей).
        \item Определить класс нового объекта путем голосования среди этих \(k\) соседей.
    \end{enumerate}
\end{enumerate}

\textbf{3. Поясните принцип выбора количества соседних объектов, по которым определяется принадлежность целевого объекта к результирующему классу.}
Выбор \(k\) является задачей оптимизации модели. Слишком малое \(k\) (например, 1) делает модель сложной и склонной к переобучению. Слишком большое \(k\) делает модель слишком простой и может привести к игнорированию реальных закономерностей в данных (недообучение). Оптимальное значение \(k\) подбирается экспериментально, обычно с использованием кросс-валидации, выбирается то \(k\), при котором достигается наилучшая обобщающая способность (максимальная точность на валидационных данных).

\textbf{4. В чем заключается метод парзеновского окна?}
Метод парзеновского окна — это непараметрический способ оценки плотности распределения. В контексте классификации он обобщает идею kNN. Вместо фиксированного числа соседей \(k\), рассматривается окно фиксированного радиуса \(h\). Классифицируемый объект относится к классу, который имеет наибольшее количество объектов внутри этого окна. Если в окно не попадает ни одного объекта, классификация невозможна. Параметр \(h\) требует оптимизации.

\textbf{5. Поясните принцип метода потенциальных функций.}
Метод потенциальных функций можно рассматривать как взвешенную версию метода парзеновского окна. Каждый объект обучающей выборки создает вокруг себя «потенциал» — функцию, убывающую с расстоянием. Потенциал класса в некоторой точке пространства вычисляется как сумма потенциалов всех объектов этого класса. Новый объект относится к тому классу, чей суммарный потенциал в этой точке максимален.

\textbf{6. Назовите, какие параметры оптимизируют в методах kNN?}
В классическом методе kNN основным оптимизируемым параметром является количество соседей \(k\). Дополнительно можно оптимизировать:
\begin{itemize}
    \item \textbf{Метрику расстояния:} выбор между евклидовым, манхэттенским, расстоянием Чебышева и другими.
    \item \textbf{Веса соседей:} можно учитывать вклад соседа в голосование с весом, обратно пропорциональным расстоянию до него (чем ближе сосед, тем больше его "голос").
    \item \textbf{Способ предобработки данных:} тип масштабирования признаков (нормализация, стандартизация) также влияет на результат.
\end{itemize}

\section{Листинг программы}
\begin{lstlisting}[language=Python, caption=Полный код лабораторной работы]
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.neighbors import KNeighborsClassifier
from sklearn.preprocessing import StandardScaler

# 1. Загрузка и подготовка данных
df = pd.read_csv('abalone.csv')

# Создание целевой переменной - возрастные категории
def age_category(rings):
    if rings <= 9:
        return 0
    elif rings < 16:
        return 1
    else:
        return 2

df['AgeCat'] = df['Rings'].apply(age_category)

# Преобразование признака Sex в dummy-переменные
df = pd.get_dummies(df, columns=['Sex'], prefix=['Sex'], drop_first=False)

# Определение признаков и целевой переменной
feature_columns = ['Length', 'Diameter', 'Height', 'Whole weight',
                   'Shucked weight', 'Viscera weight', 'Shell weight',
                   'Sex_F', 'Sex_I', 'Sex_M']
X = df[feature_columns].values
y = df['AgeCat'].values

# 2. Масштабирование данных
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 3. Функция для предсказания с заданным K
def predict_with_k(k_value, new_data):
    model = KNeighborsClassifier(n_neighbors=k_value)
    model.fit(X_scaled, y)
    new_data_scaled = scaler.transform(new_data.reshape(1, -1))
    prediction = model.predict(new_data_scaled)
    return prediction[0]

# Пример использования (задание K пользователем)
print("--- Задание K пользователем ---")
k_user = 5
example_index = 10
example_data = X[example_index]
predicted = predict_with_k(k_user, example_data)
print(f"Для объекта с индексом {example_index}, истинный класс: {y[example_index]}, предсказанный (K={k_user}): {predicted}\n")

# 4. Оценка hold-out для разных K и test_size
k_values = range(1, 51)
test_sizes = np.arange(0.1, 0.51, 0.05)
holdout_scores = np.zeros((len(k_values), len(test_sizes)))

for i, k in enumerate(k_values):
    for j, test_size in enumerate(test_sizes):
        X_train, X_test, y_train, y_test = train_test_split(X_scaled, y, test_size=test_size, random_state=42, stratify=y)
        model = KNeighborsClassifier(n_neighbors=k)
        model.fit(X_train, y_train)
        holdout_scores[i, j] = model.score(X_test, y_test)

# Визуализация hold-out
plt.figure(figsize=(12, 8))
sns.heatmap(holdout_scores.T, xticklabels=k_values[::5], yticklabels=[f"{ts:.2f}" for ts in test_sizes],
            annot=True, fmt='.3f', cmap='viridis', cbar_kws={'label': 'Accuracy'})
plt.xlabel('Количество соседей (K)')
plt.ylabel('Доля тестовой выборки (test_size)')
plt.title('Точность классификации (Hold-out) в зависимости от K и test_size')
plt.tight_layout()
plt.savefig('holdout_heatmap.png', dpi=150)
plt.show()

# 5. Оценка Cross-Validation для разных K и folds
cv_folds = range(3, 11)
cv_scores = np.zeros((len(k_values), len(cv_folds)))

for i, k in enumerate(k_values):
    for j, cv in enumerate(cv_folds):
        model = KNeighborsClassifier(n_neighbors=k)
        scores = cross_val_score(model, X_scaled, y, cv=cv, scoring='accuracy')
        cv_scores[i, j] = np.mean(scores)

# Визуализация cross-validation
plt.figure(figsize=(12, 8))
sns.heatmap(cv_scores.T, xticklabels=k_values[::5], yticklabels=list(cv_folds),
            annot=True, fmt='.3f', cmap='plasma', cbar_kws={'label': 'Average Accuracy'})
plt.xlabel('Количество соседей (K)')
plt.ylabel('Количество блоков (folds)')
plt.title('Средняя точность классификации (Cross-Validation) в зависимости от K и folds')
plt.tight_layout()
plt.savefig('cv_heatmap.png', dpi=150)
plt.show()
