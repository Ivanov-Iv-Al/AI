\section{Теоретическое обоснование}

Логические методы классификации основаны на построении решающих деревьев — иерархической структуры, в которой каждый внутренний узел соответствует проверке некоторого признака, а листья — классам. Наиболее известным алгоритмом построения решающих деревьев является CART (Classification and Regression Trees), который использует критерий Джини для выбора оптимального разбиения.

\subsection{Критерий информативности}

Для выбора атрибута, по которому производится разбиение, используются различные критерии:

\begin{itemize}
    \item \textbf{Энтропийный критерий:} 
    \[
    H(S) = -\sum_{i=1}^{c} p_i \log_2 p_i
    \]
    где \(p_i\) — доля объектов класса \(i\) в выборке \(S\).

    \item \textbf{Критерий Джини:}
    \[
    Gini(S) = 1 - \sum_{i=1}^{c} p_i^2
    \]
    Чем меньше значение критерия Джини, тем более "чистым" является узел.
\end{itemize}

\subsection{Параметры решающего дерева}

Основные параметры, влияющие на построение дерева:
\begin{itemize}
    \item \texttt{max\_depth} — максимальная глубина дерева;
    \item \texttt{max\_features} — максимальное количество признаков, используемых для поиска наилучшего разбиения;
    \item \texttt{min\_samples\_split} — минимальное количество объектов для разделения узла;
    \item \texttt{min\_samples\_leaf} — минимальное количество объектов в листе.
\end{itemize}

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# Загрузка данных
df = pd.read_csv('abalone.csv')

# Создание возрастных категорий
def age_category(rings):
    if rings <= 9:
        return 0  # Молодые
    elif rings < 16:
        return 1  # Взрослые
    else:
        return 2  # Старые

df['AgeCategory'] = df['Rings'].apply(age_category)

# Преобразование категориального признака 'Sex'
df = pd.get_dummies(df, columns=['Sex'], prefix=['Sex'])

# Определение признаков и целевой переменной
feature_columns = ['Length', 'Diameter', 'Height', 'Whole weight',
                   'Shucked weight', 'Viscera weight', 'Shell weight',
                   'Sex_F', 'Sex_I', 'Sex_M']

X = df[feature_columns].values
y = df['AgeCategory'].values

print(f"Размер матрицы признаков X: {X.shape}")
print(f"Распределение классов: {np.bincount(y)}")
\end{lstlisting}

\textbf{Шаг 6: Поиск оптимальных параметров с помощью GridSearchCV.}

\begin{lstlisting}[language=Python, caption=GridSearchCV для поиска оптимальных параметров]
# Определение сетки параметров
param_grid = {
    'max_depth': range(3, 16),
    'max_features': range(2, 11),
    'min_samples_split': [2, 5, 10],
    'min_samples_leaf': [1, 2, 4]
}

# Создание и настройка GridSearchCV
grid_search = GridSearchCV(
    DecisionTreeClassifier(random_state=42),
    param_grid,
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    verbose=1
)

# Выполнение поиска
grid_search.fit(X_scaled, y)

# Вывод результатов
print("\n=== Результаты GridSearchCV ===")
print(f"Лучшие параметры: {grid_search.best_params_}")
print(f"Лучшая точность (CV): {grid_search.best_score_:.4f}")

# Визуализация результатов GridSearchCV
results_df = pd.DataFrame(grid_search.cv_results_)
print("\nТоп-5 комбинаций параметров:")
print(results_df[['param_max_depth', 'param_max_features', 'param_min_samples_split', 
                  'param_min_samples_leaf', 'mean_test_score']].sort_values('mean_test_score', ascending=False).head(5))
\end{lstlisting}

\textbf{Пояснение величин:}
\begin{itemize}
    \item \textbf{fold} — количество блоков в кросс-валидации (в данном коде cv=5). Данные разбиваются на 5 частей, обучение и проверка выполняются 5 раз.
    \item \textbf{candidate} — каждая комбинация параметров из сетки param\_grid, для которой вычисляется качество модели.
    \item \textbf{fit} — процесс обучения модели на обучающих данных.
\end{itemize}

\section{Контрольные вопросы}

\textbf{1. Поясните принцип построения дерева решений.}

Дерево решений строится рекурсивно путем разделения обучающей выборки на подмножества. На каждом шаге выбирается признак и пороговое значение, обеспечивающие наилучшее разделение (максимальное уменьшение неопределенности). Процесс продолжается до достижения критерия остановки (максимальная глубина, минимальное количество объектов в узле и т.д.). Полученное дерево представляет собой набор правил вида "ЕСЛИ признак < значение ТО перейти в левую ветвь ИНАЧЕ в правую".

\textbf{2. Укажите статистическое определение информативности.}

Статистическая информативность признака определяется как способность признака разделять объекты разных классов. Количественно это может быть оценено, например, с помощью критерия Фишера:
\[
F = \frac{\text{межклассовая дисперсия}}{\text{внутриклассовая дисперсия}}
\]
Чем больше значение F, тем более информативен признак для разделения классов.

\textbf{3. Поясните энтропийное определение информативности.}

Энтропийная информативность основана на понятии энтропии Шеннона. Признак считается информативным, если его использование приводит к максимальному уменьшению энтропии (неопределенности) при разбиении:
\[
IG(S, A) = H(S) - \sum_{v \in Values(A)} \frac{|S_v|}{|S|} H(S_v)
\]
где \(H(S)\) — энтропия до разбиения, \(H(S_v)\) — энтропия после разбиения по значению \(v\) признака \(A\).

\textbf{4. Что такое многоклассовая информативность? Для чего она применяется?}

Многоклассовая информативность — это обобщение понятия информативности на случай более чем двух классов. Она применяется для оценки качества признаков в задачах классификации с тремя и более классами. Наиболее распространенными подходами являются:
\begin{itemize}
    \item Усреднение попарных информативностей (one-vs-one);
    \item Использование многоклассовых критериев, таких как энтропия Джини или многоклассовый коэффициент корреляции.
\end{itemize}

\textbf{5. Поясните назначение и алгоритм бинаризации количественных признаков.}

Бинаризация количественных признаков преобразует непрерывные значения в бинарные (0/1). В контексте решающих деревьев это необходимо для построения правил вида "признак < порог". Алгоритм бинаризации:
\begin{enumerate}
    \item Сортировка значений признака.
    \item Рассмотрение всех возможных порогов между соседними значениями из разных классов.
    \item Выбор порога, максимизирующего критерий информативности (например, прирост информации).
\end{enumerate}

\textbf{6. Поясните порядок поиска закономерностей в форме конъюнкций.}

Поиск закономерностей в форме конъюнкций (правил вида "A1 И A2 И ... И Ak → класс") в решающих деревьях реализуется следующим образом:
\begin{enumerate}
    \item Каждый путь от корня до листа дерева представляет собой конъюнкцию условий.
    \item Правила извлекаются путем обхода дерева.
    \item Для каждого листа формируется правило: ЕСЛИ (условия на пути) ТО класс листа.
    \item Полученные правила могут быть упрощены (удаление избыточных условий) и ранжированы по качеству.
\end{enumerate}
